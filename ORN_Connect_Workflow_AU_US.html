All supplier contracts, rate status, API integration phases, booking pipeline, and payment tracking run through Zoho. Which Zoho modules do we use?

⚡ API-connected suppliers need no manual request. If a supplier is on direct API to Nexus — bookings confirm automatically, no manual sales support step, no 72hr wait. Zoho tracks each supplier's API status and flags issues. Which suppliers should be API-first?

05Nexus Loading — Philippines team process

The Philippines team handles all Nexus product loading — receiving rate sheets from UK contracting via Zoho, loading product onto Nexus, and confirming accuracy to UK for QC. ⚡ If a supplier has direct API connection to Nexus — no rate loading is required. API suppliers book automatically with zero manual intervention. What is the Philippines team responsible for?

⚡ API-connected suppliers = zero loading work. When a hotel, attraction, or transfer operator connects directly via API, availability and pricing are live in real-time. No rate sheet. No manual load. No QC check on rates. Philippines team role for these suppliers is API configuration and monitoring only.

Quality Standards

Every product checked against UK rate sheet (Zoho attachment) before marking complete. Price, dates, inclusions, release conditions, and operator details must match exactly. Philippines marks Zoho task "Loaded — awaiting UK QC" — never "Live" until UK approves.
